High Availability Architecture  

The index replication feature synchronizes search indexes in multiple instances of SearchBlox across multiple physical or virtual machines. The Indexing Server makes the indexes available for replication, and the Search Servers get the search indexes from the Indexing Server. This integrated communication ensures your data is always replicated and fail-safe.

High Performance Engineering 

SearchBlox is engineered from the ground up to be a 100% Java solution. This enables you to leverage the performance and the scalability of java technologies for your benefit. The SearchBlox architecture consists of several internal sub-systems designed for optimal performance, flexibility and scalability. SearchBlox is also tested in high security deployments for vulnerabilities.
